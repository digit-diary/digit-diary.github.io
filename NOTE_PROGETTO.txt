================================================================================
  PANORAMICA
================================================================================

  Applicazione web single-file (index.html) per la gestione del personale
  del Casino Lugano SA. Permette di registrare eventi, redigere rapporti
  giornalieri, inviare note tra colleghi, generare moduli disciplinari
  con firma digitale e QR code, consultare statistiche, generare riepilogo
  mensile PDF, e gestire visibilita di pagine/funzioni per operatore.

  Tecnologie utilizzate:
  - HTML/CSS/JavaScript (tutto in un unico file)
  - Supabase REST API (database PostgreSQL cloud)
  - jsPDF + AutoTable (esportazione PDF)
  - Chart.js (grafici statistiche)
  - Flatpickr (calendario date)
  - Groq API (AI: llama-3.3-70b-versatile per testo,
    llama-3.2-90b-vision-preview per immagini)
  - qrcode-generator (QR code su PDF moduli)

  - Password hashate con SHA-256
  - Codice di recupero generato automaticamente
  - Prevenzione flash login al refresh (script inline pre-body)
  - Controllo deve_cambiare_pwd anche dopo refresh pagina
  - Credenziali Supabase offuscate con XOR encoding
  - Row Level Security (RLS) su tabelle sensibili
  - Operazioni auth via RPC server-side (SECURITY DEFINER)
  - pwd_hash e recovery_code mai accessibili via REST API


================================================================================
  DATABASE SUPABASE - TABELLE
================================================================================

  registrazioni
    id, nome, tipo, testo, data, created_at, operatore,
    importo, valuta, reparto, modificato_da

  rapporti_giornalieri
    id, data_rapporto, turno, sup_note, cassa_note, sala_note,
    n_assegni, prelievi, assenze, operatore, note_extra (JSONB),
    updated_at, created_at
    UNIQUE (data_rapporto, turno)
    note_extra contiene: differenze_cassa + eventuali campi custom

  note_fissate
    registrazione_id (PK), fissata_at

  scadenze
    id, registrazione_id, titolo, descrizione, data_scadenza,
    completata, completata_da, created_at

  impostazioni
    chiave (PK), valore
    Chiavi usate: tipi_personalizzati, colori_override, operatori_lista,
                  campi_rapporto_extra, tipi_nascosti, campi_nascosti,
                  tipi_ordine, campi_ordine, campi_label_override,
                  tipi_rinominati, visibilita

  note_colleghi
    id, da_operatore, a_operatore, messaggio, letta,
    nascosta_dest, nascosta_mitt, gruppo_id, created_at

  operatori_auth
    nome (PK), pwd_hash, recovery_code, ruolo,
    deve_cambiare_pwd, created_at

  collaboratori
    id, nome (UNIQUE), attivo, created_at

  moduli
    id, tipo (allineamento/apprezzamento/rdi), collaboratore,
    resp_settore, data_modulo, dati (JSONB), operatore, created_at
    dati contiene: non_conformita, obiettivo, scadenza, livello,
                   descrizione, osservazioni, qr_code,
                   firma_digitale, firma_resp, firma_collab

  log_attivita
    id, operatore, azione, dettagli, created_at

  Config visibilita (in impostazioni, chiave 'visibilita'):
    JSON con chiavi = nome funzione/pagina, valori:
    - "tutti" (default, omesso dal JSON)
    - "admin"
    - "nascosto"
    - {"tipo":"selezionati","operatori":["Nome1","Nome2"]}
