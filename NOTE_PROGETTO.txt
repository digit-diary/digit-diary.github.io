================================================================================
  PANORAMICA
================================================================================

  Applicazione web single-file (index.html) per la gestione del personale
  del Casino Lugano SA. Permette di registrare eventi, redigere rapporti
  giornalieri, inviare note tra colleghi e consultare statistiche.

  Tecnologie utilizzate:
  - HTML/CSS/JavaScript (tutto in un unico file)
  - Supabase REST API (database PostgreSQL cloud)
  - jsPDF + AutoTable (esportazione PDF)
  - Chart.js (grafici statistiche)
  - Flatpickr (calendario date)


================================================================================
  FUNZIONALITA PRINCIPALI
================================================================================

  1. DIARIO REGISTRAZIONI
     - Registrazione eventi per collaboratore (nome, tipo, descrizione)
     - Tipi di evento: Attivita, Richiesta, Errore, Ammonimento Verbale,
       Malattia, Apprezzamento, Nota, Altro (+ tipi personalizzati)
     - Per il tipo "Errore": campi extra Reparto, Importo e Valuta
     - Filtri: per persona, tipo, periodo (Dal/Al), ricerca testo
     - Note fissate (pin) per evidenziare registrazioni importanti
     - Profilo collaboratore con cronologia e grafici
     - Esportazione CSV e PDF con layout professionale

  2. RAPPORTO GIORNALIERO
     - Calendario mensile con navigazione mese/anno
     - Due turni per giorno: PRESTO e NOTTE
     - Campi predefiniti: Supervisore, Cassa, Sala, Incasso Assegni,
       Prelievi, Assenze (+ campi personalizzati dall'admin)
     - Salvataggio automatico silenzioso (autosave ogni 1.2 secondi)
     - Auto-rilevamento malattie dal campo assenze
     - Esportazione PDF con layout identico al sito:
       tabella per giorno con colonne Campo/PRESTO/NOTTE
     - Esportazione CSV con stessa struttura
     - Filtro Dal/Al per esportare un periodo specifico

  3. NOTE COLLEGHI
     - Invio messaggi a uno o piu operatori contemporaneamente
     - Checkboxes per destinatari + opzione "Tutti"
     - Note multi-destinatario raggruppate per gruppo_id
     - Notifiche in tempo reale (polling ogni 15 secondi)
     - Badge con contatore note non lette sul tab
     - Popup automatico note non lette al login
     - Modifica messaggio inviato (agisce su tutto il gruppo)
     - Annullamento invio (elimina per tutti i destinatari)
     - Blocco modifica/annullamento quando la nota e stata letta
     - Eliminazione indipendente: mittente e destinatario gestiscono
       le proprie note separatamente
     - Conferma prima di eliminare

  4. STATISTICHE
     - Riepilogo: totale registrazioni, collaboratori, errori, malattie
     - Grafico mensile (ultimi 12 mesi) con registrazioni ed errori
     - Grafico a ciambella per distribuzione tipi
     - Grafico per giorno della settimana
     - Top 15 collaboratori per numero di registrazioni
     - Tabella dettagliata collaboratori x tipi con totali
     - Tabella errori con importi (CHF/EUR) e reparti

  5. SCADENZE E PROMEMORIA
     - Creazione scadenze collegate a registrazioni
     - Campanella con badge nel header (numero scadenze attive)
     - Dropdown con lista scadenze imminenti
     - Completamento scadenza con nome operatore e data
     - Scadenze completate visibili con barrato

  6. IMPOSTAZIONI (ADMIN)
     - Gestione tipi di evento: aggiungere, rimuovere, cambiare colore
       (anche i tipi predefiniti possono essere nascosti/ripristinati)
     - Gestione campi rapporto giornaliero: aggiungere, rimuovere
       (anche i campi predefiniti possono essere nascosti/ripristinati)
     - Gestione operatori: creare, rimuovere, resettare password
     - Cambio password master
     - Tema chiaro/scuro

  7. MODULI DISCIPLINARI
     - Tre moduli PDF generabili dall'app:
       1. Allineamento: rilevamento non conformita + colloquio
       2. Apprezzamento: colloquio di apprezzamento
       3. RDI: rapporto disciplinare interno (I° e II° livello)
     - Campi compilabili con dati pre-compilati (data, resp. settore)
     - Generazione PDF con layout professionale
     - Firme e distribuzione automatiche


================================================================================
  SISTEMA DI AUTENTICAZIONE
================================================================================

  Login a due livelli:

  ADMIN (password master)
  - Accesso completo a tutte le funzionalita
  - Puo creare e rimuovere operatori
  - Puo cambiare colori tipi e gestire campi rapporto
  - Puo resettare password operatori
  - Puo cambiare la password master
  - Nelle impostazioni vede solo "Cambia password master"

  OPERATORE (password personale)
  - Accesso alle funzionalita operative (diario, rapporto, note, statistiche)
  - Al primo accesso viene richiesto di cambiare la password temporanea
  - Puo cambiare la propria password
  - Non puo creare/rimuovere operatori ne modificare tipi/campi
  - Nelle impostazioni vede solo "Cambia la mia password"

  Sicurezza:
  - Password hashate con SHA-256
  - Codice di recupero generato automaticamente
  - Prevenzione flash login al refresh (script inline pre-body)
  - Controllo deve_cambiare_pwd anche dopo refresh pagina


================================================================================
  DATABASE SUPABASE - TABELLE
================================================================================

  registrazioni
    id, nome, tipo, testo, data, created_at, operatore, importo, valuta, reparto

  rapporti_giornalieri
    id, data_rapporto, turno, sup_note, cassa_note, sala_note,
    n_assegni, prelievi, assenze, operatore, note_extra,
    updated_at, created_at
    UNIQUE (data_rapporto, turno)

  note_fissate
    registrazione_id (PK), fissata_at

  scadenze
    id, registrazione_id, titolo, descrizione, data_scadenza,
    completata, completata_da, created_at

  impostazioni
    chiave (PK), valore

  note_colleghi
    id, da_operatore, a_operatore, messaggio, letta,
    nascosta_dest, nascosta_mitt, created_at

  operatori_auth
    nome (PK), pwd_hash, recovery_code, ruolo,
    deve_cambiare_pwd, created_at

  collaboratori
    id, nome (UNIQUE), attivo, created_at

  moduli
    id, tipo (allineamento/apprezzamento/rdi), collaboratore,
    resp_settore, data_modulo, dati (JSONB), operatore, created_at
